Developer Getting Started

Setting Up Your Development Environment

Let's build Emily's system step by step:

bash

Install the Buildly CLI


npm install -g @buildly/cli


Create a new project for Emily's system  


buildly create patient-data-system


cd patient-data-system


Initialize with healthcare template


buildly init --template healthcare

This automatically sets up HIPAA-compliant defaults, security middleware, and audit logging - exactly what Emily needs.

Backend Development with Buildly CLI

Generate the core services for patient data management:

bash

Create the patient service with security patterns


buildly generate service patient-service --security hipaa


Add the Patient model with encrypted fields


buildly add model Patient --fields "name:encrypted,ssn:encrypted,dob:date"


Generate secure API endpoints


buildly add endpoint patients --auth required --audit true


buildly add endpoint patients/emergency --auth emergency-override

Each command follows the security patterns Emily requires, with automatic encryption, audit trails, and compliance validation.

Frontend Development with React Template

Create a responsive, accessible interface:

bash

Generate React frontend with healthcare UI components


buildly create-frontend react --template healthcare


Add patient dashboard components  


buildly generate component PatientDashboard --features "search,filters,emergency-access"


buildly generate component PatientRecord --features "view,edit,audit-history"

The healthcare template includes pre-built components that meet accessibility standards and integrate seamlessly with your backend APIs.

Integration and Deployment

Connect your components using Buildly's orchestration:

bash

Configure services to work together


buildly connect patient-service frontend --auth-flow oauth2


buildly add middleware audit-logger --all-services


buildly add middleware encryption --sensitive-data


Deploy to staging environment


buildly deploy staging --compliance-check

The platform automatically handles service discovery, API routing, and compliance validation. Reference https://docs.buildly.io/deployment for production considerations.
